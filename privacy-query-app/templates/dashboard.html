How It Works
🔒 Privacy Protection Process
  1. You submit a query - Include any information you want
  2. PII Detection - System detects sensitive data:
    • Names (PERSON)
    • Organizations (ORG)
    • Locations (GPE, LOC)
    • Emails & Phones
    • ID Numbers (Aadhaar, PAN, SSN)
    • Passwords & API Keys
    • Credit/Debit Cards
    • And more...
  3. Masking - PII replaced with placeholders
    • john@example.com → [EMAIL]
    • +91 9876543210 → [PHONE]
    • John Doe → [NAME]
  4. Risk Calculation - Privacy risk score computed
  5. AI Processing - Masked query sent to Llama 3 via Ollama
  6. Unmasking - Response restored with original values
  7. Results Display - You see the final, unmasked response
⚙️ Risk Score Calculation

Formula: Risk Score = Σ (weight × count)

Weights by Data Type:

Password/API Key 50
Aadhaar/PAN/SSN 40-45
Credit/Debit Card 35
Email/Phone 30
Name/Org/Location 20
IP Address 10

  • ✅ User Privacy: PII never sent to external AI
  • ✅ Data Security: Sensitive info stays with you
  • ✅ Compliance: Reduces GDPR/privacy law risks
  • ✅ Trust: Users know their data is protected
